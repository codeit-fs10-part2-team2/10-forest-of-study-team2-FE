# API Endpoints: JS vs JSON 파일 비교

## JavaScript 파일 (현재 사용)

### 장점
1. **동적 경로 생성**: 함수를 사용하여 파라미터를 받아 동적으로 경로 생성 가능
   ```javascript
   GET_BY_ID: (id) => `/api/studies/${id}`
   ```

2. **타입 안정성**: TypeScript 사용 시 타입 체크 및 자동완성 지원

3. **IDE 지원**: 자동완성, 리팩토링, 네비게이션 등 강력한 IDE 기능 활용 가능

4. **런타임 유연성**: 조건부 로직, 계산된 값 등 런타임에 동적으로 결정 가능

5. **코드 재사용**: 공통 로직을 함수로 추출하여 재사용 가능

6. **유지보수성**: 변경사항이 있을 때 한 곳에서 수정 가능

### 단점
1. **번들 크기**: JavaScript 코드로 번들에 포함되어 약간의 크기 증가

2. **정적 분석 제한**: 일부 정적 분석 도구가 함수 호출을 추적하기 어려울 수 있음

3. **다른 언어 호환성**: 다른 언어/프로젝트에서 직접 사용하기 어려움

## JSON 파일

### 장점
1. **단순성**: 구조가 단순하고 직관적

2. **다른 언어 호환성**: Python, Java 등 다른 언어에서도 쉽게 파싱 가능

3. **설정 파일로 적합**: 순수한 데이터 구조로 관리하기 쉬움

4. **정적 데이터**: 변경이 거의 없는 정적 데이터에 적합

5. **번들 크기**: JSON은 파싱만 하면 되므로 약간 더 가벼울 수 있음

### 단점
1. **동적 경로 불가**: 함수를 포함할 수 없어 파라미터가 필요한 경로를 표현하기 어려움
   ```json
   // 불가능: 함수를 포함할 수 없음
   {
     "GET_BY_ID": "(id) => `/api/studies/${id}`"  // 문자열로만 저장 가능
   }
   ```

2. **타입 체크 불가**: 런타임에만 오류 발견 가능

3. **IDE 지원 제한**: 자동완성, 타입 체크 등 제한적

4. **런타임 파싱 필요**: 사용 시 문자열을 파싱하거나 템플릿 리터럴로 변환해야 함

5. **유지보수 어려움**: 동적 경로를 사용하려면 별도의 헬퍼 함수 필요

## 결론

**현재 프로젝트에서는 JavaScript 파일이 더 적합합니다.**

이유:
- API 엔드포인트에 파라미터가 필요한 경로가 많음 (studyId, habitId, emojiId 등)
- 함수를 사용하여 동적으로 경로를 생성할 수 있어 코드가 깔끔함
- TypeScript 도입 시 타입 안정성 확보 가능
- IDE 지원으로 개발 생산성 향상

JSON 파일은 다음과 같은 경우에 적합:
- 정적 경로만 있는 경우
- 다른 언어/프로젝트와 공유해야 하는 경우
- 설정 파일로 관리하고 싶은 경우
